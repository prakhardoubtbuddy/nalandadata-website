Overview

Overview

DrishtiTable is built on 1,421 expert-annotated tables from 9 S. Chand academic textbooks, spanning statistics, finance, accounting, and engineering. It targets what frontier models miss on domain-specific, structurally complex tables.

01

Expert Annotation

1,421 tables hand-annotated to ground-truth HTML — headers, merged cells, hierarchy — by domain experts.

02

Domain Diversity

9 source books, 6 subject families, 5 table types — from simple grids to complex financial statements.

03

Held-out Benchmark

135-table frozen test set, never seen in training, scored with the standard TEDS metric. Verified, public.

04

Open & Reproducible

Public sample + eval script on Hugging Face (Apache-2.0). Run any model on the test set — no form required.

The approach

Methodology.

How DrishtiTable was built and the model trained — four reproducible steps.

1

Curate & annotate

1,421 tables sourced from 9 S. Chand textbooks, each manually annotated to ground-truth HTML with semantic structure (thead/tbody, colspan/rowspan, formatting).

2

Split & freeze

Split 1,141 train / 145 val / 135 test. The test set is content-hashed and held private to prevent contamination.

3

Fine-tune (QLoRA)

Qwen2.5-VL-7B fine-tuned with 4-bit QLoRA via Unsloth on a single A100 — about 35 minutes of compute.

4

Evaluate (TEDS)

Every model scored with the canonical TEDS scorer against expert ground truth. Top failure modes for frontier models: content divergence, row-count mismatch, and flattened merged headers.
